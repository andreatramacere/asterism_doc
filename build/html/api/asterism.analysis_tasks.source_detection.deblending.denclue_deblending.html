Module: denclue_deblending¶

Overview¶

This modules provides the implementation of the deblending based on the DENCLUE algorithm. The basic idea is to partition a cluster according to the clusters of attractors found by the DENCLUE. The resulting sub-clusters are analyzed and accepted or rejected according to some user-specified criteria, and a final deblended list of source clusters is returned. The Task is implemented by the DoDENCLUEDeblendingTask class, and the Task algorithm is implemented by the do_denclue_deblending() top-level function. This function implements a two step algorithm, as shonw in the graph below:

  • the configuration of the DENCLUE algorithm, and the denlcue partitioning of the clusters, done by the do_denclue_of_cluster(),
  • the validations of the resulting sub-clusters performed by the get_deblended_cluster_list()

This function implements the top-level algorithm for the Denclue deblending:

  • Each cluster in the cluster_list is partitioned by the .do_denclue_of_cluster function.
  • The resulting denclue sub-clusters are processed by the .get_deblended_cluster_list that validates the sub-clusters, and returns the final list of accepted subclusters.
